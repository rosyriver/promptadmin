请帮我设计并实现一个个人用的「Prompt Case Library」网页应用。

这是一个用于管理图片和视频 Prompt 资产的个人收藏系统。用户不是从零创作 Prompt，而是收集、整理、复刻和复用看到的优秀 Prompt 案例。每条资产都必须绑定一个图片或视频预览，并支持一键复制 Prompt。

请采用单页应用设计，不要设计多个页面，不要做左侧栏，不要做后台管理风格。

# 一、产品定位

产品名称：Prompt Case Library

产品目标：
帮助用户把优秀的图片 / 视频 Prompt 案例保存成个人资产库。

核心使用流程：

1. 用户看到一个优秀图片或视频案例
2. 上传对应图片或视频
3. 粘贴 Prompt
4. 添加标签
5. 可选填写来源链接、原作者、模型、笔记
6. 保存为一个 Prompt Case
7. 以后通过搜索、标签筛选、媒体预览快速找回
8. 一键复制 Prompt，用于复刻或改造

核心原则：

* 媒体预览是主角
* Prompt 是资产内容
* 标签是主要组织方式
* 一键复制是最高频操作
* 收录流程必须轻量
* 页面要高密度显示更多卡片
* 不要做复杂项目管理、团队协作、版本管理、AI 自动生成等功能

# 二、视觉风格

请采用「紧凑 Pinterest 风 + Figma / 创意工具风」的混合设计。

整体气质：

* 像一个个人视觉灵感库
* 像一个轻量创作工具
* 像 Pinterest 的图片墙，但更紧凑、更工具化
* 像 Figma 的操作体验，但不要变成复杂设计软件
* 不要像传统后台管理系统
* 不要像数据仪表盘
* 不要像 Notion 文档库

视觉关键词：

* 高密度媒体网格
* 图片 / 视频优先
* 顶部搜索与标签筛选
* 轻量工具栏
* 胶囊标签
* 右侧详情抽屉
* 悬浮操作按钮
* 干净、克制、现代
* 长期使用不疲劳

风格要求：

1. 页面整体可以使用浅色或深色，但推荐浅灰 / 米白 / 低饱和背景，突出媒体内容
2. 卡片要像紧凑版 Pinterest，而不是大留白作品集
3. 卡片之间间距要小，但不能拥挤
4. 标签使用 Figma 式小型 chip
5. 顶部筛选栏像创意工具的控制条
6. 右侧详情抽屉像 Figma 属性面板，清晰、轻量、可编辑
7. 使用细边框、轻微阴影、中等圆角
8. 强调色只使用一种，例如蓝紫色、薄荷绿或电光蓝
9. 不要使用大面积渐变
10. 不要使用夸张插画
11. 不要让文字抢走媒体的注意力

推荐视觉方向：

* 背景：#F6F6F4 或 #FAFAF8
* 主文本：#18181B
* 次级文本：#71717A
* 边框：#E4E4E7
* 卡片背景：#FFFFFF
* 强调色：#7C3AED 或 #2563EB
* 标签背景：#F4F4F5
* 标签选中：强调色浅色背景 + 强调色文字
* 圆角：12px 到 16px
* 卡片阴影：非常轻，不要厚重

如果使用暗色风格：

* 背景：#0F1014
* 卡片：#181A20
* 边框：#2A2D35
* 文本：#F4F4F5
* 次级文本：#A1A1AA
* 强调色：#8B5CF6 或 #22D3EE

# 三、页面整体结构

请实现为单页应用，结构如下：

顶部固定工具区：

* 第一行：应用名称、搜索框、+ 收录按钮
* 第二行：类型筛选和标签筛选

主体区域：

* 高密度媒体卡片网格
* 默认展示所有 Prompt Case
* 不要左侧筛选栏
* 不要默认展示表格

右侧详情抽屉：

* 点击卡片后打开
* 展示媒体预览、Prompt、标签、来源、笔记
* 支持复制 Prompt

收录弹窗：

* 点击「+ 收录」打开
* 用于上传媒体、粘贴 Prompt、打标签

# 四、顶部工具区

顶部区域应该固定在页面顶部，滚动时保持可见。

第一行包含：

1. 左侧应用名称：Prompt Case Library
2. 中间搜索框，占主要宽度
   placeholder：「搜索 Prompt / 标签 / 来源 / 作者 / 模型」
3. 右侧主按钮：「+ 收录」

搜索框要求：

* 支持搜索 Prompt 内容
* 支持搜索标签
* 支持搜索来源链接
* 支持搜索原作者
* 支持搜索模型
* 搜索和标签筛选可以同时生效

第二行包含快捷筛选：

类型筛选：

* 全部
* 图片
* 视频

常用标签筛选：

* 电影感
* 国风
* 产品图
* 小红书
* 雨夜
* 人物
* 极简
* 真实摄影
* 插画
* 更多

筛选交互：

1. 标签筛选支持多选
2. 多个标签筛选采用 AND 逻辑，也就是同时满足
3. 标签点击后进入选中态
4. 再次点击取消选中
5. 类型筛选为单选
6. 「更多」点击后弹出更多标签面板
7. 更多面板中可以显示全部标签、模型、来源平台等低频筛选项

# 五、主体卡片网格

主体使用紧凑媒体卡片网格。

不要使用传统瀑布流。
不要使用表格作为默认视图。
不要在卡片上显示标题。
不要在卡片上显示状态字段。

桌面端目标：

* 一行尽量显示 5 到 7 张卡片
* 卡片宽度建议 180px 到 240px
* 使用 CSS grid 自适应布局
* gap 控制在 12px 到 16px
* 一屏尽量展示更多资产

卡片内容只包含：

1. 图片或视频封面
2. 类型角标：Image / Video
3. 最多 3 个标签
4. 如果标签超过 3 个，显示 +N
5. 「复制 Prompt」按钮
6. 更多操作按钮「⋯」

图片卡片：

* 显示图片预览
* 图片使用 object-cover
* 保持统一卡片比例，例如 4:5 或 1:1

视频卡片：

* 默认显示视频封面
* 右上角显示 Video 角标
* 中间可以显示轻量播放图标
* 鼠标悬停时自动播放
* 鼠标移开后暂停，并恢复封面状态
* 视频默认静音
* 视频不需要显示控制条

卡片交互：

1. 点击「复制 Prompt」直接复制 Prompt
2. 复制成功后显示 toast：「已复制 Prompt」
3. 点击卡片非按钮区域打开右侧详情抽屉
4. 鼠标悬停卡片时显示轻微浮起效果
5. 更多按钮「⋯」可显示编辑、删除等操作

卡片视觉：

* 圆角 14px 左右
* 细边框
* 轻微阴影
* 媒体区域占卡片 75% 以上
* 标签和按钮区域尽量紧凑
* 不要让文字区域过高

# 六、收录弹窗

点击「+ 收录」后打开居中 Modal。

标题：收录 Prompt Case

表单字段：

必填：

1. 上传媒体

   * 支持图片和视频
   * 支持拖拽上传
   * 支持点击选择文件
   * 上传后自动识别 image 或 video
   * 上传后显示预览

2. Prompt

   * 多行文本框
   * 用于粘贴完整 Prompt
   * 支持较长文本
   * placeholder：「粘贴你的 Prompt」

3. 标签

   * 支持预设标签选择
   * 支持自由输入标签
   * 每条资产最多 8 个标签
   * 卡片上最多显示 3 个标签

可选：

1. 来源链接
2. 原作者
3. 模型

   * 可选：Midjourney、GPT Image、Sora、Runway、Kling、Pika、其他
   * 支持自定义输入
4. 简短笔记

表单要求：

1. 不要设置标题字段
2. 不要设置状态字段
3. 不要要求用户填写来源
4. 不要要求用户填写作者
5. 不要让收录流程复杂
6. 保存按钮：「保存」
7. 另一个按钮：「保存并继续收录」
8. 保存后关闭弹窗，并将新案例插入卡片网格
9. 点击「保存并继续收录」后清空表单，继续保留弹窗打开

# 七、右侧详情抽屉

点击卡片后，从右侧滑出详情抽屉。

抽屉宽度：

* 桌面端：420px 到 520px
* 移动端：全屏

抽屉顶部：

1. 关闭按钮
2. 大图或视频预览
3. 类型信息：Image / Video
4. 如果填写了模型，显示模型名称

抽屉页签：

1. Prompt
2. 标签
3. 来源
4. 笔记

默认打开 Prompt 页签。

Prompt 页签：

* 显示完整 Prompt 文本
* 提供主按钮：「复制 Prompt」
* 提供编辑按钮
* Prompt 文本区域要适合长文本阅读
* 复制成功显示 toast：「已复制 Prompt」

标签页签：

* 显示当前标签
* 支持添加标签
* 支持删除标签
* 支持预设标签和自由标签
* 最多 8 个标签

来源页签：

* 显示来源链接
* 显示原作者
* 显示模型
* 显示收藏日期
* 所有字段都可以为空
* 来源链接如果存在，应可点击打开

笔记页签：

* 显示用户笔记
* 支持编辑
* 笔记可以记录：

  * 为什么收藏
  * 关键提示词
  * 可替换元素
  * 复刻注意事项
  * 适合用途
